# Plans

Use this directory for implementation plans, rollout plans, migration plans, and project planning documents.

## Naming

Name files with the date first, followed by a specific kebab-case description, and ending with the model name:

```text
YYYY-MM-DD-specific-plan-name-<model>.md
```

Examples:

```text
2026-04-28-worktree-layout-migration-plan-gpt5-4.md
2026-04-28-release-workflow-cleanup-plan-sonnet4-6.md
```

The name should make the plan's goal and authoring model clear without opening the file.

## Project Tracking

Use `PROJECT_STATUS.md` to track the current project state and change log.

## Archived Plans

Use `archived/` for completed, superseded, or paused plans.
