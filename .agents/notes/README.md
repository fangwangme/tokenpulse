# Notes

Use this directory for research notes, session learnings, implementation context, and reference material that should stay with the project.

## Naming

Name files with the date first, followed by a specific kebab-case description, and ending with the model name:

```text
YYYY-MM-DD-specific-topic-<model>.md
```

Examples:

```text
2026-04-28-pricing-cache-review-gpt5-4.md
2026-04-28-quota-parser-notes-sonnet4-6.md
```

The name should make the note's subject and authoring model clear without opening the file.

## Archived Notes

Use `archived/` for historical notes that are no longer active but still useful as project context.
